# CHARAN (차란)

**브라우저에서 돌아간다는 게 안 믿기는 럭셔리카 컨피규레이터**

> 조하민 (BSSM 3113) · 프론트엔드 포트폴리오 프로젝트

---

## 1. 프로젝트 정의

### 한 줄

> _브라우저에서 돌아간다는 게 안 믿기는 럭셔리카 컨피규레이터._

### 핵심 가치

CHARAN은 **"웹에서 이게 된다고?"** 라는 반응을 목표로 하는 R3F 기반 자동차 컨피규레이터다.
포토리얼리스틱한 카페인트 셰이더와 시네마틱 라이팅으로, 브랜드 공식 사이트급 비주얼을
Next.js + React Three Fiber만으로 구현한다.

### 네이밍

**차란 (CHARAN)** — "차" + "찬란". 찬란한 차.
한국어 어감과 영문 표기가 자연스럽게 공존하는 네이밍 (쏘카, 차즘 계열).

## 2. 목적 & 성공 지표

### 1순위 목적

**프론트엔드 이직/인턴용 포트폴리오.**
기술력을 한눈에 증명하는 단일 작품. 이력서에서 "이거 하나만 보세요" 라고 말할 수 있는 대표작.

### 왜 이 프로젝트인가

- 프론트엔드 포트폴리오 대부분이 Todo/블로그/쇼핑몰 CRUD에 머무름
- 3D 웹 (R3F)은 국내 프론트 시장에서 희소 스킬
- 디자인 + 프론트 + 3D 수학 + 셰이더를 동시에 증명 가능
- 한 장의 스크린샷으로 임팩트 전달 가능 (채용담당자는 5초 본다)

## 3. 타겟 사용자 & 시나리오

프론트엔드 포트폴리오의 특수성: **한 명의 유저가 아니라, 서로 다른 세 관점의 독자가 같은 화면을 본다.**
각 독자가 다른 부분에서 감동해야 한다.

### 타겟 A: 채용담당자 / 개발팀 리더

**첫 5초 임팩트 독자.**

- 포트폴리오 링크 클릭 → 로딩 → "오." → 스크롤/클릭 → "얘 뽑자"
- 이력서 리뷰 중 수십 개 링크 중 하나. 집중력 없음.
- **필요한 것**: 첫 화면에서 즉각적인 시각적 임팩트. 설명 없이 통과.

**시나리오**:

1. 링크 클릭
2. 로딩 스크린에서 브랜딩 인지 (3초 이내)
3. 마이바흐가 스튜디오 조명 아래 등장
4. 자동으로 한 바퀴 돌거나 카메라가 시네마틱하게 이동
5. 색상 패널 발견 → 한두 개 눌러봄 → 페인트 전환 애니메이션에 놀람
6. 탭 닫고 다음 지원자로... 가 아니라 북마크

### 타겟 B: 일반 사용자

**진짜 서비스처럼 쓰고 싶은 독자.**

- "어 이거 실제 마이바흐 사이트인가?" 착각
- 이것저것 눌러보고 조합해보고 공유하고 싶음
- **필요한 것**: UX 완성도. 버튼이 예쁘고 반응이 부드러움. 에러 없음. 모바일에서도 됨.

**시나리오**:

1. 차를 드래그로 돌려봄
2. 색 바꿔봄
3. 휠 바꿔봄
4. 배경 바꿔봄 (스튜디오 → 선셋 → 도심)
5. "오 이거 저장/공유되나?" → 링크 공유 기능 발견
6. 친구한테 링크 보냄

### 타겟 C: 개발자 커뮤니티

**기술 깊이 독자.**

- 트위터/블로그에서 "어떻게 만들었지?" 가 궁금함
- 소스 코드 볼 가능성 높음
- **필요한 것**: 기술적 깊이가 드러나는 디테일. 커스텀 셰이더, 퍼포먼스 최적화, 코드 품질.

**시나리오**:

1. 트위터 클립 보고 유입
2. 라이브 데모 열어봄
3. devtools 열어서 드로우콜, FPS 확인
4. GitHub 링크 찾음 → README 정독
5. 기술 블로그 포스트 읽음 → 리트윗

### 3축이 동시에 요구하는 것

- **첫 5초 임팩트** (타겟 A)
- **완성도 높은 UX** (타겟 B)
- **기술적 디테일** (타겟 C)

셋 다 만족시키지 못하면 포트폴리오로 실패. **"대충 멋있기만 한 3D 웹"** 은 흔함.

## 4. MVP 스코프

### 판단 기준

> **"이 기능이 '웹에서 이게 된다고?' 반응에 기여하는가?"**

답이 No면 전부 v2로.

### IN (MVP에 들어감)

#### 4.1 차량 모델

- **Mercedes-Maybach S-Class 2022** 1대
- 출처: Sketchfab @mpgs.studio (CC Attribution 기준 예상, 라이선스 재확인 필요)
- Draco 압축 적용 완료 (77MB → 5.2MB, 93% 감소)

#### 4.2 카페인트 셰이더 ⭐ (프로젝트 심장)

- `MeshPhysicalMaterial` 베이스 + `onBeforeCompile` 확장
- 구성 요소:
  - **Base color** (메탈릭 페인트 기본색)
  - **Clearcoat** (투명 코팅층, 마이바흐 시그니처)
  - **Metallic flake** (미세 금속 입자, 각도에 따라 반짝임)
  - **Two-tone paint** (위/아래 색 분리 — 마이바흐 고유 디자인)
- 이 셰이더 하나가 프로젝트 차별점의 80%

#### 4.3 라이팅 & 환경

- HDRI 환경맵 3종 (Poly Haven CC0)
  - **Studio** (전문 촬영 스튜디오, 기본값)
  - **Sunset** (황혼, 따뜻한 톤)
  - **Night City** (도심 야경, 네온 반사)
- 환경맵 전환 시 부드러운 lerp (1초 블렌딩)
- ACES 톤매핑

#### 4.4 카메라 시스템

- 시네마틱 프리셋 앵글 5개:
  1. **Front 3/4** — 기본 히어로 샷
  2. **Side profile** — 실루엣 감상
  3. **Rear 3/4** — 테일램프 강조
  4. **Wheel close-up** — 휠 디테일
  5. **Front grille** — 마이바흐 그릴 클로즈업
- 앵글 간 전환은 부드러운 lerp
- 자유 회전: OrbitControls (damping 적용)
- 줌 제한 (모델이 화면 밖으로 나가지 않음)

#### 4.5 인터랙션

- **컬러 변경**: 6~8색 (솔리드 / 메탈릭 / 펄 카테고리)
- **투톤 on/off**: 싱글 컬러 ↔ 투톤 페인트 전환
- **휠 변경**: 3종 (마이바흐 기본 / 모노블럭 / 멀티스포크)
- **환경 전환**: 3종 HDRI 토글
- **카메라 앵글**: 프리셋 버튼
- 전환은 모두 애니메이션 (instant 금지)

#### 4.6 UI

- 좌측 또는 하단 미니멀 패널
- 색상 스와치, 아이콘 버튼, 슬라이더 최소한
- Tailwind + 커스텀 디자인 시스템
- **디자인 톤**: 마이바흐 공식 사이트급 미니멀/럭셔리
- 모바일: 바텀시트로 변형

#### 4.7 로딩 경험

- 로딩 스크린 = 첫인상의 50%
- 브랜딩 ("CHARAN" 로고) + 프로그레스 인디케이터
- 모델 로드 완료 시 시네마틱 등장 (카메라 줌인 + 페이드인)

#### 4.8 퍼포먼스

- Draco 압축 GLB
- 모바일 터치 지원
- 타겟: Lighthouse Performance 90+
- 목표 FPS: 데스크톱 60fps, 모바일 30fps+

- **차량 모델**: Sketchfab (@mpgs.studio, Mercedes-Maybach 2022)

- **Vercel**

### 개발 툴

- **bun** — 패키지 매니저 / 런타임
- **ESLint + Prettier**
- **Git + GitHub**

- **Next.js App Router**: 라우팅 단순하지만 SEO/메타태그 유리 (포트폴리오 공유 시 OG 이미지 중요)

- **Tailwind (vs CSS Modules)**: 디자인 이터레이션 속도
- **bun (vs npm/pnpm)**: 빠름, 개인 선호

## 6. 차별화 포인트

R3F 컨피규레이터는 이미 많다. CHARAN이 다른 이유:

### 6.1 커스텀 카페인트 셰이더

대부분의 R3F 자동차 프로젝트는 `MeshStandardMaterial` 기본값 또는 단순 MeshPhysicalMaterial만 쓴다.
CHARAN은 **metallic flake + clearcoat + two-tone** 을 직접 구현한 셰이더를 탑재한다.
→ 기술 블로그 1편 분량의 깊이.

### 6.2 투톤 페인트 (마이바흐 시그니처)

다른 컨피규레이터에서 거의 볼 수 없는 기능.
차량 모델도 마이바흐로 선정한 이유 중 하나.
→ "이 프로젝트만의 특별한 한 가지".

### 6.3 시네마틱 카메라 전환

즉각적 위치 변경 대신 부드러운 lerp + easing.
각 앵글이 하나의 "샷(shot)" 처럼 느껴지게.
→ 영상 편집 감각 (Premiere Pro, After Effects 경험 활용).

### 6.4 로딩 경험까지 디자인

대부분의 3D 웹 프로젝트는 로딩이 "검은 화면 + 스피너".
CHARAN은 로딩부터 브랜딩 경험의 일부.
→ 디자인 감각 증명.

### 6.5 모바일 퍼포먼스

3D 웹 프로젝트 대부분 모바일 버림.
CHARAN은 드레이코 압축 + LOD + 적극적 최적화로 모바일에서도 돌아감.
→ "모바일에서도 돌아감" 자체가 차별점.

- [ ] 투톤 on/off

## 9. v2 확장 계획

MVP 배포 후, 반응을 보면서 다음 중 선택 확장:

### Tier 1 (반응 좋으면 바로)

- **차량 2대째 추가** — 롤스로이스 팬텀 또는 부가티 시론. 여기부터가 진짜 "컨피규레이터".
- **URL 공유 기능** — `?color=midnight&wheel=02&env=sunset` 형태. 친구한테 자기 조합 공유.
- **스크린샷 저장** — 현재 뷰를 이미지로 다운로드.

### Tier 2 (기술 챌린지)

- **문/트렁크 개폐** — GLTF 애니메이션 트리거.
- **인테리어 뷰** — 카메라가 차 안으로 진입.
- **야간 모드** — 헤드라이트/테일라이트 켜짐 (emissive).

### Tier 3 (실험적)

- **WebXR AR 모드** — 실제 공간에 차 배치.
- **비디오 녹화** — 사용자 인터랙션을 영상으로 뽑아주기.
- **다국어 지원** — 영/한/일.

## 부록 A. 프로젝트 철학

> **"대충 멋있는 게 아니라, 진짜로 잘 만든 것."**

시간이 여유롭다는 건 퀄리티 기준을 타협할 이유가 없다는 뜻이다.
CHARAN은 "학생 프로젝트치고는 잘했다" 가 아니라 "이거 스튜디오에서 만든 건 줄 알았다" 를 목표로 한다.

채용담당자가 5초 만에 판단하는 세상에서, 포트폴리오는 **한 방의 강한 인상**이 열 개의 평범한 프로젝트를 이긴다.
CHARAN은 그 한 방이 되어야 한다.
